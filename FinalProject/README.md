# Projecto final Ironhack

## Objetivo

La tarea es crear un modelo que prediga las decisiones de determinados clientes ante determinadas ofertas, además de comprender los datos empleados para construir el modelo.

## Descripción de los datos

El conjunto de datos consta de información sobre 18.000 clientes. Estas son las definiciones de los puntos de datos proporcionados:

●	Numero de cliente: Este es un número secuencial asignado a los clientes (esta columna está oculta y excluida; este identificador único no se utilizará directamente).

●	Oferta aceptada: El cliente aceptó (Yes) o rechazó (No) la oferta. 

●	Recompensa: el tipo de programa de recompensa que se ofrece para la tarjeta.

●	Tipo de correo: Carta o postal.

●	Nivel de income: Low, Medium, o High.

●	Cuentas abiertas: Cuántas cuentas que no son de tarjeta de crédito tiene el cliente.

●	Protección contra sobregiros: ¿Tiene el cliente protección contra sobregiros en su(s) cuenta(s) corriente(s)? (Yes o No).

●	Calificación crediticia: Low, Medium, o High.

●	Tarjetas en posesión: El número de tarjetas que el cliente tiene con el banco.

●	Casas apropiadas: El número de casas a nombre del cliente.

●	Tamaño del hogar: número de personas en la familia.

●	Poseer su casa: ¿El cliente es dueño de su casa? (Yes o No).

●	Saldo promedio: Saldo promedio de la cuenta (en todas las cuentas a lo largo del tiempo). Q1, Q2, Q3 y Q4.

●	Saldo: Saldo promedio de cada trimestre en el último año.
